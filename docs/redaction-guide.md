# 脱敏指南

在公开分享、二次分发或演示本项目时，建议确认仓库中不包含真实客户资料。

## 建议替换或排除出公开仓库的内容

- 公司名称。
- 客户名称。
- 供应商名称。
- 银行名称。
- 审计机构名称。
- 项目金额。
- 合同编号。
- 税号。
- 统一社会信用代码。
- 地址。
- 联系人。
- 电话。
- 身份证号。
- 银行账号。
- 盖章页。
- 签字页。
- 审计报告签章。
- 征信报告。

## Office 文件风险

`.docx`、`.xlsx` 可能包含隐藏信息：

- 修订记录。
- 批注。
- 作者信息。
- 文档属性。
- 隐藏 sheet。
- 隐藏行列。
- 外部链接。
- 缓存数据。

公开发布前，建议检查文件属性、修订记录和隐藏 sheet，避免遗留隐含信息。

## 样例数据规则

样例数据建议全部使用虚构信息。

推荐使用：

- 客户A、客户B、客户C。
- 供应商A、供应商B、供应商C。
- 银行A、银行B。
- 示例企业A、示例企业B。

不建议直接以真实项目材料改名后发布，因为金额结构、客户集中度和科目组合仍可能构成商业信息。

# 脱敏指南

发布到 GitHub 前，必须确认仓库中没有真实客户资料。

## 必须删除或替换

- 公司名称。
- 客户名称。
- 供应商名称。
- 银行名称。
- 审计机构名称。
- 项目金额。
- 合同编号。
- 税号。
- 统一社会信用代码。
- 地址。
- 联系人。
- 电话。
- 身份证号。
- 银行账号。
- 盖章页。
- 签字页。
- 审计报告签章。
- 征信报告。

## Office 文件风险

`.docx`、`.xlsx` 可能包含隐藏信息：

- 修订记录。
- 批注。
- 作者信息。
- 文档属性。
- 隐藏 sheet。
- 隐藏行列。
- 外部链接。
- 缓存数据。

发布前应打开文件属性、修订记录和隐藏 sheet 人工检查。

## 样例数据规则

样例数据必须完全虚构。

推荐使用：

- 客户A、客户B、客户C。
- 供应商A、供应商B、供应商C。
- 银行A、银行B。
- 江苏星河电子科技有限公司。

不要从真实项目直接改名后发布，因为金额结构、客户集中度和科目组合仍可能构成商业信息。

# 财务重点科目底稿

## 适用场景

用于中文尽调、授信、租赁、担保人或承租人财务分析底稿中，整理资产负债表重点科目。常见用户说法包括：

- “整理重要科目 sheet”
- “把占比超过 5% 的资产科目、超过 3% 的负债权益科目列出来”
- “披露二级科目明细”
- “应收账款/应付账款/其他应收款/其他应付款列前五大”
- “根据审计报告和科目余额表补充重点科目”
- “解释报表调整/重分类是什么意思”
- “写财务尽调前先把主要科目清单整理好”

本技能通常在 `$financial-lessee-due-diligence` 前使用。它负责生成或复核财务底稿中的 `重要科目` sheet；正式尽调正文再根据该 sheet 选择披露内容。

若需要英文版或脚本模板，可同时参考 `$key-subject-analysis`。

## 核心原则

1. 以财务报表 Excel 的资产负债表为控制总数。
   不直接用科目余额表总账余额替代报表列示金额。

2. 先筛选，再展开。
   默认资产科目按总资产 `5%` 筛选；负债及权益科目按负债与权益总额 `3%` 筛选。若用户指定某些往来科目，即使不达阈值，也要作为“补充披露”列示。

3. 任一重点期间触发阈值，即入选。
   通常重点比较最近审计年度期末和最新一期期末。若 2024 年末或 2025 年末任一期间超过阈值，该科目均应进入重要科目清单。用户工作簿若已经设置更多期间，可尊重其口径，但必须复核筛选逻辑。

4. 审计年度优先取审计报告附注。
   审计报告附注已有二级明细、前五大单位、账龄、坏账准备、固定资产分类、借款担保方式等披露时，优先使用附注。

5. 最新一期优先从科目余额表提取。
   从科目编码、科目名称、核算维度、期末借方/贷方余额中提取二级明细和前五大往来。没有科目余额表时，再使用最新一期半年报、管理报表或用户已整理的 Excel 明细。

6. 明细表必须勾到报表列示值。
   每个重点科目表都要有 `合计` 行，合计金额应等于资产负债表列示金额。若明细加总和报表列示不一致，列 `报表调整/重分类`。

7. 底稿可以详细，正式尽调要取舍。
   `重要科目` sheet 可以保留筛选汇总、双期间前五大、口径说明等底稿信息；正式尽调正文不应把这些底稿内容原样全搬进去。

## 输入材料

优先使用以下材料：

- 三年审计报告，尤其是财务报表附注。
- 最新一期科目余额表或科目明细表。
- 用户整理的财务信息工作簿，包含资产负债表、利润表、现金流量表、财务指标。
- 已有尽调模板或已有 `重要科目` sheet。

若材料不完整：

- 没有最新一期科目余额表时，使用最新一期半年报或管理报表中的附注/明细。
- 没有前五大明细时，不要凭空编；可保留上年审计附注前五大，并注明来源期间。
- 没有审计附注明细时，可用科目余额表拆分，但要标注口径来自余额表。

## 推荐输出结构

在目标 Excel 的 `重要科目` 或类似 sheet 中，按以下顺序组织：

1. `入选科目及占比汇总`
   列示类型、科目、上年末账面价值、上年末占比、最新期账面价值、最新期占比、入选依据。

2. `占资产总额5%以上的资产科目明细`
   每个科目一张小表，列示项目、上年末余额、最新期余额、合计。

3. `占负债与权益总额3%以上的负债及权益科目明细`
   每个科目一张小表，列示项目、上年末余额、最新期余额、合计。

4. `未达阈值但按要求补充披露的往来科目`
   例如其他应收款未达 5%，但用户要求披露前五大时，在此处补充。

5. `往来款前五大客户/公司`
   应收账款、其他应收款、应付账款、其他应付款分别列示。底稿中可以同时保留上年末和最新期两组前五大，便于比较和复核。

6. `报表调整/重分类口径说明`
   对每个调节项解释原因和使用口径。

## 筛选口径

资产类科目：

- 分母为同一期间资产负债表 `资产总计`。
- 筛选阈值为 `5%`。
- 计算公式：该资产科目账面价值 / 资产总计。
- 任一重点期间超过 5%，即入选。

负债及权益类科目：

- 分母为同一期间资产负债表 `负债和所有者权益总计`，通常等于资产总计。
- 筛选阈值为 `3%`。
- 计算公式：该负债或权益科目账面价值 / 负债和所有者权益总计。
- 任一重点期间超过 3%，即入选。

补充披露科目：

- 用户指定披露的往来科目，即使未达阈值，也要进入 `未达阈值但按要求补充披露的往来科目`。
- 常见补充披露科目包括其他应收款、其他应付款、应收账款、应付账款、预付款项、合同负债等。

不要机械筛选以下项目：

- 小计、合计、流动资产合计、非流动资产合计、流动负债合计、所有者权益合计等标题或合计行。
- 星号指标行、分析指标行、财务指标表中的派生项目。
- 明显不是资产负债表项目的管理统计项。

## 审计报告和科目余额表取数

审计报告适合提取：

- 2022、2023、2024 等审计年度的附注明细。
- 应收账款账龄、坏账准备、前五大客户。
- 其他应收款性质、坏账准备、前五大单位。
- 存货分类和跌价准备。
- 固定资产原值、累计折旧、减值准备、类别。
- 借款种类、担保方式、抵押质押、利率区间。
- 长期应付款、租赁负债、售后回租、未确认融资费用。
- 股本、资本公积、未分配利润等权益项目。

科目余额表适合提取：

- 最新一期二级科目余额。
- 最新一期往来核算维度。
- 最新一期银行存款、其他货币资金、短期借款、长期借款、应付票据、其他应付款等明细。

科目余额表常见列结构：

- `科目编码`
- `科目名称`
- `核算维度名称`
- `期末余额-借方`
- `期末余额-贷方`

处理规则：

- 资产类用期末借方余额；备抵科目用负数扣减。
- 负债及权益类用期末贷方余额。
- 二级科目优先取直接下级科目编码，例如 `1601.01`、`1602.01`。
- 前五大往来取核算维度名称，去掉 `/公司/声表` 等无关后缀。
- 前五大按正向余额从大到小排序。
- 若存在负数明细导致前五大正向余额合计高于净额，需要写复核备注。

## 常见科目拆分口径

货币资金：

- 常见明细：库存现金、银行存款、其他货币资金。
- 注意银行承兑汇票保证金、信用证保证金、冻结资金、受限资金。
- 若余额表明细合计与报表列示存在小额差异，可列 `报表调整/重分类`。

应收账款：

- 常见明细：应收账款账面余额、坏账准备、报表调整/重分类、合计。
- 坏账准备在底稿中建议用负数列示，或用 `减：坏账准备` 明确为扣减项。
- 前五大客户按最新一期核算维度或附注排序。

其他应收款：

- 注意报表列示金额可能远低于科目余额表往来维度金额，常见原因包括内部往来抵消、报表重分类、坏账准备。
- 若余额表显示分公司、子公司、关联方大额往来，要在备注中提示口径差异。

存货：

- 常见明细：原材料、在产品/生产成本、自制半成品、库存商品、发出商品、低值易耗品、委托加工物资、跌价准备。
- 存货跌价准备作为扣减项处理。

固定资产：

- 常见明细：房屋及建筑物、机器设备、电子设备、运输设备、办公设备、其他设备。
- 若从余额表提取，按固定资产原值减累计折旧、减值准备后列示。
- 注意固定资产抵押、售后回租、融资租赁限制。

短期借款：

- 常见明细：信用借款、保证借款、抵押借款、质押借款、保理、票据贴现、供应链融资、应计利息。

一年内到期的非流动负债：

- 常见明细：一年内到期的长期借款、一年内到期的租赁负债、一年内到期的长期应付款。
- 注意与长期借款、长期应付款的重分类勾稽。

长期借款：

- 常见明细：信用、保证、抵押、质押、担保抵押借款。
- 若最新一期长期借款明细含一年内到期部分，应列 `减：一年内到期重分类` 以勾稽至报表列示。

长期应付款：

- 常见明细：售后回租款、融资租赁款、应付融资租赁款、未确认融资费用。
- 未确认融资费用作为扣减项处理。

应付账款：

- 常见明细：应付采购款、暂估应付、购建长期资产款、其他。
- 前五大供应商按最新一期核算维度或附注排序。

其他应付款：

- 常见明细：其他应付款、预提费用、专项应付款、员工报销、押金保证金、客户保证金。
- 注意是否存在实质融资属性或大额关联往来。

权益类：

- 实收资本/股本：按股本、实收资本列示。
- 资本公积：资本溢价、其他资本公积。
- 未分配利润：若亏损较大，应提示累计亏损和盈利修复。

## 前五大底稿规则

底稿中的前五大可以比正式尽调更详细。

底稿允许：

- 同时列示上年末和最新期前五大，便于比较两期客户或供应商变化。
- 使用列组：`上年单位名称 / 上年余额 / 上年占比 / 最新期单位名称 / 最新期余额 / 最新期占比`。
- 对最新期前五大与报表列示金额存在口径差异的情况写复核备注。

底稿必须：

- 前五大按正向余额从大到小排序。
- 明确金额单位。
- 明确占比口径，是占报表列示余额、账面余额、还是余额表核算维度余额。
- 若最新期前五大来自余额表，注明其代表余额表核算维度，不一定等同于报表附注口径。

与正式尽调衔接时：

- 正式尽调只披露最新一期前五大。
- 只有最新一期没有前五大资料时，才使用上一年审计附注前五大，并在文字中说明来源。
- 从底稿双期间前五大表转入正式尽调时，只取最新期三列：`单位名称 / 最新期末余额 / 占比`。
- 不要把 `上年单位名称`、`上年余额`、`上年占比` 等列带入正式尽调正文。
- 正式尽调中的前五大表格应放在该科目的文字说明之后，而不是放在文字说明之前。

## 报表调整/重分类说明

`报表调整/重分类` 不是实际二级科目，也不是客户或供应商名称。它用于解释：

```text
报表列示金额 = 明细提取金额 + 报表调整/重分类
```

出现原因通常包括：

- 审计或报表层面的重分类调整。
- 内部往来抵消。
- 坏账准备、减值准备、折旧摊销列示口径不同。
- 一年内到期的非流动负债重分类。
- 未确认融资费用扣减。
- 科目余额表科目和资产负债表项目不是一一对应。
- 金额单位从元转换为万元后的尾差。

底稿中可以设置独立的 `报表调整/重分类口径说明` 区域，逐项解释差异。

正式尽调中不要把该独立口径说明区域原样搬入正文。若需要解释，应在对应科目的文字分析中简要说明，例如：

```text
该科目明细合计与报表列示金额存在差异，主要系内部往来抵消或报表重分类影响，最终以资产负债表列示金额为准。
```

## 与承租人财务尽调的衔接

当本技能服务于 `$financial-lessee-due-diligence` 时，按以下口径交接：

1. `入选科目及占比汇总` 只作为尽调科目清单检查表，不写入正式尽调。
2. `占资产总额5%以上的资产科目明细` 和 `占负债与权益总额3%以上的负债及权益科目明细` 是正式尽调主要科目分析的基础。
3. `未达阈值但按要求补充披露的往来科目` 若用户要求披露，应写入正式尽调。
4. `往来款前五大客户/公司` 在正式尽调中只取最新一期。
5. `报表调整/重分类口径说明` 不作为正式尽调独立章节。
6. 正式尽调的科目顺序应为：科目标题、科目明细表、文字分析、前五大表格（如有）。
7. 若正式尽调发现 Excel 重要科目表格比 Word 报告多，应补齐报告，不要用一句话概括替代。

## 复核清单

交付前至少检查：

- 阈值筛选结果是否与资产负债表重新计算一致。
- 资产类是否按总资产 5% 筛选。
- 负债及权益类是否按负债与权益总额 3% 筛选。
- 任一重点期间触发阈值的科目是否均已入选。
- 不达阈值但用户指定披露的科目是否单独标注。
- 每个明细表 `合计` 是否等于资产负债表列示金额。
- 若明细和报表不一致，是否列示 `报表调整/重分类`。
- 所有 `报表调整/重分类` 是否有解释。
- 前五大单位是否按余额或附注排序。
- 前五大占比口径是否清晰。
- 最新一期有前五大资料时，是否已优先使用最新一期。
- 若服务于正式尽调，是否提醒正式报告只取最新一期前五大。
- 是否保留原工作簿格式、公式和其他 sheet。
- 保存前是否备份原 Excel 或另存补充版。
- 若 Excel 被打开导致文件锁定，先保存补充版，待用户关闭后再覆盖原文件。

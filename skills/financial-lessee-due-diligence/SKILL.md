# 承租人财务尽调

## 核心原则

以用户整理的财务信息 Excel 作为结构化主源；审计报告、科目余额表、半年报和 OCR 文本作为补充来源，用于核对附注、解释变化、补足最新一期明细。不要改动源文件。先复制尽调模板或真实样式参考文件，再只在复制件上写作和排版。

正式写尽调前，必须先使用或复核本地 `$financial-key-subjects` 技能。该技能用于生成 `重要科目`/`重点科目` sheet，按资产占比 5%、负债及权益占比 3% 筛选重点科目，并整理二级科目明细、往来前五大和报表调整/重分类勾稽。承租人财务尽调中的“主要科目情况分析”应以该重点科目结果作为强制清单。

优先联用以下技能：

- `$financial-key-subjects`：先生成或复核重要科目清单，筛选资产 5%、负债及权益 3% 重点科目，整理二级明细、往来前五大和报表口径调节。
- `$xlsx`：读取三年一期报表、财务指标、重要科目 sheet 和万元口径表。
- `$docx`：复制模板、保持 Word 样式、检查表格和段落格式。
- `$pdf`：读取审计报告附注、受限资产、借款、担保、融资租赁等披露。

## 常见输入

用户通常会提供：

- 最近三年审计报告。
- 最新一期科目余额表、科目明细表；少数情况下不提供科目余额表，而是提供半年报或最新一期报告。
- 用户整理的承租人财务信息工作簿，通常包含三年一期资产负债表、利润表、现金流量表、财务指标和 `重要科目` sheet。
- 尽调模板目录，通常包含空白尽调模板、真实尽调样式参考 Word、财务信息录入模板等。

## 总体工作流

1. 盘点文件，识别模板、真实样式参考、财务信息工作簿、审计报告、最新一期科目余额表/半年报和 OCR 文本目录。
2. 读取真实样式参考，记录章节顺序、表格结构、字体、字号、行距、缩进、标题加粗方式、表格边框、表头底纹、列宽和行高。
3. 读取财务信息工作簿。若工作簿中已有万元口径表，直接使用万元口径，不要从元口径重复换算。
4. 使用 `$financial-key-subjects` 生成或复核 `重要科目` sheet。若需要新增或重算 `重要科目`，先复制工作簿或另存补充版，不要覆盖用户源文件。
5. 复制模板或真实样式参考到新 Word 文件。用户强调格式时，优先用真实样式参考作为格式母版，再替换公司名称、期间、表格和正文。
6. 按模板写入三年一期报表、财务指标和分析段落。主要科目情况分析必须来自 `重要科目` 清单，不能漏科目。
7. 用审计报告和科目余额表补充说明科目构成、受限资产、借款担保、融资租赁、往来单位、坏账准备、折旧摊销、减值和重分类。
8. 交付前做格式复核、数据复核、重要科目完整性复核和文字残留复核，确认 Word 能正常打开，源文件没有被修改。

## 重点科目前置流程

写正式尽调前，先执行以下前置动作：

1. 查看用户财务信息工作簿是否已有 `重要科目` 或类似 sheet。
2. 若已有，仍需用 `$financial-key-subjects` 的规则复核筛选结果：
   - 资产类科目：以资产负债表总资产为分母，筛选占总资产超过 5% 的资产科目。
   - 负债及权益类科目：以负债与权益总额为分母，筛选占负债及权益总额超过 3% 的负债和权益科目。
   - 通常以最近一年末和最新一期为重点；任一期间触发阈值的科目均应进入清单。若工作簿已按更多期间筛选，尊重工作簿口径并复核逻辑。
3. 若没有 `重要科目` sheet，或 sheet 明显不完整，先用 `$financial-key-subjects` 在复制件或补充文件中生成。
4. 重要科目明细必须以财务报表列示金额为控制总数，不得直接用科目余额表总账余额替代报表列示金额。
5. 审计年度优先使用审计报告附注中的二级明细、账龄、坏账准备、固定资产分类、借款分类、前五大等披露。
6. 最新一期优先使用科目余额表或科目明细表提取二级明细和往来前五大；没有最新明细时，再退回使用最新半年报或上年审计附注。
7. 明细表合计必须勾稽至资产负债表列示金额。若明细加总与报表列示不一致，可以在科目明细表中保留 `报表调整/重分类` 行。

`重要科目` sheet 推荐包含以下底稿内容：

- `入选科目及占比汇总`：用于检查哪些科目入选，不直接写入正式尽调正文。
- `占资产总额5%以上的资产科目明细`：每个资产科目一张小表。
- `占负债与权益总额3%以上的负债及权益科目明细`：每个负债/权益科目一张小表。
- `未达阈值但按要求补充披露的往来科目`：例如其他应收款未达 5%，但用户要求披露前五大时列入。
- `往来款前五大客户/公司`：应收账款、其他应收款、应付账款、其他应付款等。
- `报表调整/重分类口径说明`：作为底稿解释，不作为正式尽调独立章节。

## 正式尽调中的重要科目取舍

`重要科目` sheet 是正式尽调“主要科目情况分析”的强制清单，但底稿内容不能原样全搬进报告。

正式尽调正文应写入：

- 入选资产类科目的明细表和文字分析。
- 入选负债、权益类科目的明细表和文字分析。
- 虽未达阈值但用户要求披露的往来科目及文字分析。
- 有往来前五大的科目，应写入最新一期前五大表格。

正式尽调正文不应写入：

- `入选科目及占比汇总` 或类似筛选汇总表。
- 独立的 `报表调整/重分类口径说明` 板块及其汇总表。
- 只服务于底稿勾稽的解释性大表。

若某个科目明细表中需要用 `报表调整/重分类` 行才能勾稽至报表列示金额，可以保留该行；但不要另外在正式尽调中单独开一个“口径说明”小节。需要解释时，在该科目的文字分析中用一两句话说明即可。

## 主要科目情况分析排版顺序

正式尽调的“主要科目情况分析”必须按科目逐项写，不要把多个科目的表格和文字混放。

每个科目的标准顺序为：

1. 科目标题，例如 `（2）应收账款：`。
2. 该科目的明细表。
3. 该科目的文字说明。
4. 若该科目有前五大，再放该科目的前五大表格。

没有前五大的科目，顺序为：

1. 科目标题。
2. 该科目的明细表。
3. 该科目的文字说明。

不要把同一科目的所有表格都放在文字说明前面。尤其是应收账款、其他应收款、应付账款、其他应付款等有前五大的科目，应为“科目明细表 -> 文字说明 -> 前五大表格”。

不要把某个科目的文字分析放到其他科目下面。例如长期借款、一年内到期的非流动负债、长期应付款应各自有标题、表格和文字说明，不要把长期借款文字放在短期借款段落里。

## 主要科目编号和顺序

正式尽调中，主要科目编号应连续、清晰、与正文出现顺序一致。

推荐顺序：

1. 资产类重点科目。
2. 补充披露的资产往来科目，例如其他应收款。
3. 负债类重点科目。
4. 权益类重点科目。

可根据模板和企业实际适当调整，但不得出现以下问题：

- 编号跳号、重复或错序。
- 科目标题无编号而夹在编号科目之间。
- 表格属于一个科目，文字却落在另一个科目下面。
- 前五大表格游离在本应所属的科目之外。

## 前五大披露规则

正式尽调中的前五大只披露最新一期。

执行规则：

- 有最新一期前五大时，只放最新一期三列：`单位名称 / 最新期末余额 / 占比`。
- 不要在正式尽调中同时放 `2024单位名称 / 2024余额 / 2024占比 / 2025单位名称 / 2025余额 / 2025占比` 这种底稿式双期间表。
- 只有最新一期没有前五大资料时，才披露上一年审计报告附注中的前五大，并在文字中说明因最新一期资料未提供，暂以最近审计年度披露。
- 前五大表格应放在该科目文字说明之后。
- 文字说明中可先概括前五大合计金额和占比，再接前五大表格。

示例顺序：

```text
（2）应收账款：
【应收账款明细表】
截至2025年末，承租人应收账款账面价值为……。2025年前五大客户合计……万元，占比……%，客户集中度较高，具体如下：
【应收账款前五大表】
```

前五大自查：

- 表头只有最新一期，不得残留 `2024单位名称`、`2024占比` 等列。
- 合计金额和占比要与文字说明一致。
- 前五大单位应按金额从大到小排序。
- 若前五大正向余额合计高于报表净额，通常说明存在负数明细、坏账准备、内部抵销或报表重分类，应在该科目文字中提示口径差异。

## 科目分析写法

每个主要科目都必须有文字分析，不能只有表格。

每个主要科目通常写清：

- 期末余额及占总资产、总负债或所有者权益比例，比例有意义时披露。
- 较上年末增加或减少的金额。
- 较上年末增加或下降的比例，变化较明显时披露。
- 余额构成、主要二级科目、主要客户/供应商/往来单位。
- 变化原因、经营含义、偿债或回款风险、后续核查事项。

对几乎没有变化的科目，不必机械分析增长率。可以写 `较上年末基本稳定`、`与上年末基本持平`、`变化不大`，再说明余额构成即可。

遇到 `报表调整/重分类` 行时，要说明最终金额以财务报表列示为准。`报表调整/重分类` 不是实际二级科目，也不是往来单位，不要把它写成客户、供应商或借款主体。

常见科目关注点：

- `货币资金`：银行存款、其他货币资金、受限资金、对短债覆盖程度。
- `应收账款`：账面余额、坏账准备、账面价值、最新一期前五大客户、客户集中度、账龄和回款风险。
- `其他应收款`：报表列示口径和科目余额表往来口径差异；内部往来、分公司往来、押金保证金、暂付款、备用金。
- `存货`：原材料、在产品/生产成本、自制半成品、库存商品、发出商品、跌价准备、周转压力。
- `固定资产/在建工程`：主要资产类型、折旧、产能和使用状态、投保情况、抵押或售后回租限制。
- `无形资产/长期待摊费用`：土地使用权、软件、装修或模具摊销、减值迹象。
- `其他非流动金融资产/长期股权投资/投资类资产`：被投资单位、账面稳定性、流动性和减值风险。
- `递延所得税资产`：形成原因、变动情况、对未来盈利的依赖。
- `短期借款`：信用借款、保理、票据贴现、供应链融资、利息、短期集中兑付压力。
- `一年内到期的非流动负债`：一年内到期的长期借款、租赁负债、售后回租款，短期偿债压力。
- `长期借款`：信用、保证、抵押、质押、担保方式、期限结构、一年内到期重分类。
- `长期应付款`：售后回租款、融资租赁款、未确认融资费用、还款计划。
- `应付账款`：最新一期前五大供应商、供应商集中度、结算周期、逾期风险。
- `其他应付款`：最新一期前五大往来、专项应付款、员工报销、客户保证金、是否存在实质融资属性。
- `实收资本/资本公积/未分配利润`：增资、股本变动、资本公积来源、累计亏损或利润留存。

## 章节写作顺序

按模板通常顺序写：

1. 主要科目情况分析。
2. 偿债能力分析。
3. 资产受限制情况分析。
4. 盈利能力分析。
5. 资产周转比率分析。
6. 现金流量分析。
7. 发展能力分析。

不要跳过资产受限制情况分析。若现有材料未完整披露受限资产，应审慎表述为：现有资料未见完整受限资产明细，仍需结合借款合同、抵押登记、融资租赁合同、银行函证、征信报告和审计报告附注进一步核实。

偿债能力、盈利能力、周转能力、现金流和发展能力分析应使用财务指标表中的指标，不要凭印象重算。指标口径异常时，先回到 Excel 和审计报告核对。

## 格式要求

格式是交付成果的一部分。样式参考 Word 是排版标杆。

- 保持样式参考的字体、字号、行距、段前段后、首行缩进、标题加粗、编号样式和段落层级。
- 保持表格样式、列宽、行高、边框、表头底纹、单元格对齐方式和数字格式。
- 可因 `重要科目` 披露需要新增表格，但新增表格必须套用样式参考的同类表格格式。
- 不要把自动生成的默认 Word 表格直接留在报告中。
- 不要随意加粗。若样式参考中的普通科目标题、表格正文、分析正文不加粗，生成文件也不得加粗。
- 表格中文通常用宋体，数字可参照样式使用 Times New Roman；最终以样式参考为准。
- 发展能力分析等后续章节也要逐段套用样式，不能只保证前几节格式正确。
- 不要在正文中残留模板占位符、Markdown 标记、样式参考公司名称、样式参考行业、样式参考期间或无关往来单位。

## 表格填报规则

- 现金流量表中的标题行，如 `一、经营活动产生的现金流量：`、`二、投资活动产生的现金流量：`、`三、筹资活动产生的现金流量：`，数值列应留空，不得填 `0`。
- 利润表、资产负债表中的类别标题行也不要随意填 `0`，除非来源表和样式参考明确将其作为数值行。
- 以万元为单位，除模板要求外，不保留不必要的小数。
- 保留坏账准备、跌价准备、减值准备、累计折旧、长期负债一年内到期重分类、未确认融资费用等项目的正负号逻辑。若简化为正数列示，应在行名写明 `减：`。
- 明细表合计应勾稽至财务报表列示金额。若明细和报表不一致，应增加或保留 `报表调整/重分类` 行，并在该科目的文字分析中说明口径。
- 往来前五大表应披露单位名称、最新期末余额、占比；占比口径要与表头和文字一致。

## 数据复核清单

交付前逐项检查：

- 输出 Word 能正常打开，源模板、样式参考、审计报告、科目余额表和财务信息工作簿未被修改。
- 最新一期标签与用户数据一致，不残留样式参考中的期间。
- 无 `XX`、`**`、`【】`、`201*` 等占位符。
- 无样式参考公司名称、样式参考行业、样式参考客户/供应商、样式参考日期等残留。
- 已使用 `$financial-key-subjects` 生成或复核重点科目清单。
- 资产类重点科目已按总资产 5% 阈值复核，负债及权益类重点科目已按负债权益总额 3% 阈值复核。
- `重要科目` sheet 要求的科目均已覆盖，筛选汇总表和独立口径说明表除外。
- 每个科目均为“科目标题 -> 科目明细表 -> 文字说明 -> 前五大表格（如有）”。
- 有前五大的科目，前五大表格只披露最新一期；只有最新一期没有资料时才披露上一年。
- 不存在 `2024单位名称`、`2024占比` 等底稿式前五大列残留。
- 现金流标题行没有填 `0`。
- 科目分析中，重要变化已写增加/减少金额和百分比；基本稳定科目未机械堆增长率。
- 表格合计与财务报表一致，或已在该科目中清楚说明报表调整/重分类口径。
- 发展能力分析、现金流分析等后半部分格式与样式参考一致。
- 受限资产、借款、担保、融资租赁和往来前五大等高风险信息已尽量用审计报告或科目余额表核对。

## 位置和数据自查方法

修改 Word 后，至少做一次结构化自查：

- 对主要科目章节，抽取每个科目的项目顺序。应收账款、其他应收款、应付账款、其他应付款等应呈现 `T-P-T`：科目明细表、文字段落、前五大表。
- 对没有前五大的科目，应呈现 `T-P`：科目明细表、文字段落。
- 对比修改前后表格数量。若只是移动表格，表格数量应不变。
- 对比修改前后表格内容。若只是调整位置，表格内容应完全一致。
- 搜索不得残留 `入选科目及占比汇总`、`报表调整/重分类口径说明`、`2024单位名称`、`2024占比`、跳号编号等。
- 随机核对前五大合计、主表合计、文字中的金额和占比是否一致。

## 交付习惯

每次修订都另存新版本，例如 `二校`、`三校`、`四校`、`五校`，不要覆盖用户已经人工校对过的版本，除非用户明确要求覆盖。若只是调整排版或表格位置，应尽量保留用户人工修改过的文字、表格数据和格式。
